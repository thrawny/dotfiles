# Nix Configuration Rules

Nix Home Manager is the sole configuration system — never edit generated files under `~`.

If a config is generated by Nix (zsh, tmux, ghostty, starship, direnv, ...), edit the module in `nix/home/`; if it's symlinked via `mkOutOfStoreSymlink` (nvim, claude, codex, pi), edit the file in `config/`. PATH entries go in `nix/home/shared/home-base.nix` (`sessionPath`).

## After changes

- Trivial changes (adding a package to a list, renaming a string): skip validation.
- Non-trivial changes (new modules, option wiring, expressions): run `just check`.

Only when requested: `just dry` (full build test, NixOS only), `just switch`.
